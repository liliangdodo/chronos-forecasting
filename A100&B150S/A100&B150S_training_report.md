# Chronos-2 训练测评报告：A100 vs B150S

## 1. 测评任务说明

本次测评用于对比 **A100** 与 **B150S** 两类 GPU 服务器在同一 Chronos-2 训练任务下的训练时间、吞吐能力、单步/单样本处理效率以及显存占用表现。

本报告依据以下文件生成：

- `A100/training_info.json`
- `A100/training_benchmark.csv`
- `B150S/training_info.json`
- `B150S/training_benchmark.csv`

两组任务的核心配置一致，主要包括：

| 项目 | A100 | B150S |
| --- | ---: | ---: |
| 模型 | Chronos-2 | Chronos-2 |
| 数据集 | CauKer10K_H3000_V20.arrow | CauKer10K_H3000_V20.arrow |
| `context_length` | 2048 | 2048 |
| `prediction_length` | 64 | 64 |
| `min_past` | 64 | 64 |
| `max_steps` | 10000 | 10000 |
| `per_device_train_batch_size` | 32 | 32 |
| `gradient_accumulation_steps` | 1 | 1 |
| `effective_global_batch_size` | 128 | 128 |
| `world_size` | 4 | 4 |
| `epoch` | 1 | 1 |

结论：本次两组实验在**训练任务规模、批大小、步数和 GPU 数量**上是可比的。

## 2. 测评环境

### 2.1 硬件环境对比

| 项目 | A100 | B150S |
| --- | --- | --- |
| GPU 型号 | NVIDIA A100-SXM4-80GB | Iluvatar BI-V150S |
| GPU 数量 | 4 | 4 |
| 单卡总显存 | 79.25 GiB | 32.00 GiB |
| 总显存 | 317.01 GiB | 128.00 GiB |

硬件上，A100 的单卡显存约为 B150S 的 **2.48 倍**，整机总显存也约为 **2.48 倍**。

### 2.2 软件环境对比

| 项目 | A100 | B150S |
| --- | --- | --- |
| Python | 3.10.12 | 3.12.11 |
| PyTorch | 2.3.0a0+6ddf5cf85e.nv24.04 | 2.7.1 |
| Transformers | 4.57.6 | 4.57.6 |
| Accelerate | 1.13.0 | 1.7.0 |
| NumPy | 1.24.4 | 1.26.4 |
| TF32 | False | False |
| BF16 | False | False |

这次更新后的实验中，A100 与 B150S 都未启用 TF32 / BF16，因此相比之前的报告，**数值精度路径已基本对齐**。  
不过 Python、PyTorch、Accelerate 版本仍不完全一致，因此结果仍然包含一定的软件栈差异。

## 3. 测评结果

### 3.1 核心训练指标

| 指标 | A100 | B150S |

| `train_samples_per_second` | 416.520 | 227.853 |
| `train_steps_per_second` | 3.254 | 1.780 |
| `train_loss` | 3.8814 | 3.8833 |
| `epoch` | 1 | 1 |
| `global_step` | 10000 | 10000 |
| `max_steps` | 10000 | 10000 |

### 3.2 时间对比

| 指标 | A100 | B150S | A100 相对 B150S |
| --- | ---: | ---: | ---: |
| 总训练时长 | 3074.377 s | 5618.593 s | **快 1.83x** |
| 每 step 耗时 | 0.3074 s | 0.5619 s | **快 1.83x** |
| 每 1000 step 耗时 | 307.44 s | 561.86 s | **快 1.83x** |

从总训练时长看，A100 完成同样 10000 step 任务的时间约为 B150S 的 **54.7%**，整体训练完成速度约为 **1.83 倍**。

### 3.3 吞吐能力对比

| 指标 | A100 | B150S | A100 相对 B150S |
| --- | ---: | ---: | ---: |
| `train_samples_per_second` | 416.520 | 227.853 | **高 1.83x** |
| `train_steps_per_second` | 3.254 | 1.780 | **高 1.83x** |
| 每 GPU `samples/s` | 104.130 | 56.963 | **高 1.83x** |
| 每 GPU `steps/s` | 0.8135 | 0.4450 | **高 1.83x** |

吞吐指标和训练总时长结论一致：在同样 world size=4、相同 batch 配置下，A100 在样本吞吐和 step 吞吐上都明显领先。

### 3.4 单步和单样本处理能力

| 指标 | A100 | B150S | A100 相对 B150S |
| --- | ---: | ---: | ---: |
| 每样本耗时 | 2.401 ms | 4.389 ms | **快 1.83x** |
| 每 step 耗时 | 307.4 ms | 561.9 ms | **快 1.83x** |

这说明在固定训练配置下，A100 不仅整体训练完成更快，而且单步调度和单样本处理延迟也更低，更适合需要频繁迭代和快速反馈的训练场景。

### 3.5 显存占用对比

#### 3.5.1 绝对显存指标

| 指标 | A100 | B150S |
| --- | ---: | ---: |
| `max_memory_allocated_mb` | 9278.379 MB | 9669.997 MB |
| `max_memory_reserved_mb` | 15212.000 MB | 9778.000 MB |
| `avg_memory_allocated_mb` | 2079.493 MB | 2050.949 MB |
| `avg_memory_reserved_mb` | 15178.346 MB | 9765.783 MB |

可以看出：

- **峰值 allocated 显存** 两者非常接近，B150S 甚至略高，说明在这组训练配置下，两边真正用于张量的峰值显存需求近似一致。
- **reserved 显存** A100 明显更高，反映出其 PyTorch 显存缓存池更大，保留的显存更多。
- **平均 allocated 显存** 两边几乎一致，均约 2.05~2.08 GB，说明训练过程中的常态张量占用接近。

#### 3.5.2 显存利用率

以单卡总显存为基准，换算显存利用率：

| 指标 | A100 | B150S |
| --- | ---: | ---: |
| 峰值 allocated / 单卡总显存 | 11.43% | 29.51% |
| 峰值 reserved / 单卡总显存 | 18.74% | 29.84% |
| 平均 allocated / 单卡总显存 | 2.56% | 6.26% |
| 平均 reserved / 单卡总显存 | 18.70% | 29.80% |

这说明：

1. 虽然两边绝对显存占用接近，但由于 A100 单卡显存更大，其**显存压力显著更低**。
2. B150S 在同一任务下已经接近 **30%** 的 reserved 利用率，而 A100 仍低于 **19%**。
3. 若后续扩大 batch、上下文长度或模型规模，A100 在显存扩展空间上更充足。

### 3.6 资源效率视角

如果将吞吐按整机总显存做归一化：

| 指标 | A100 | B150S |
| --- | ---: | ---: |
| 每总 GiB 显存的 `steps/s` | 0.0103 | 0.0139 |
| 每总 GiB 显存的 `samples/s` | 1.3139 | 1.7801 |

这里 B150S 反而略优。原因不是它单卡训练更快，而是：

- A100 提供了更多总显存资源
- 当前这组任务并没有把 A100 的显存优势充分吃满

因此：

- **若看绝对训练速度和吞吐**：A100 更强
- **若按总显存资源归一化**：B150S 的“显存利用效率”更紧凑

### 3.7 训练结果稳定性

| 指标 | A100 | B150S | 差异 |
| --- | ---: | ---: | ---: |
| `train_loss` | 3.8814 | 3.8833 | A100 低 0.00184 |

两边最终 `train_loss` 几乎一致，差异约 **0.05%**。  
这意味着在当前 10000 step 训练任务下，两类卡都能达到接近的训练损失水平，本次差异主要体现为**训练时间、吞吐和显存压力**，而不是训练收敛质量。

## 6. 结论

在本次 **4 卡、相同 Chronos-2 训练任务、相同步数和相同有效 batch** 的测试中：

1. **A100 在绝对训练速度上更优**
   - 总训练时间约为 B150S 的 **54.7%**
   - 样本吞吐和 step 吞吐均约为 B150S 的 **1.83 倍**

2. **A100 在单步和单样本处理上更高效**
   - 每 step 耗时更低
   - 每样本耗时更低
   - 更适合需要快速迭代的训练任务

3. **两类卡的实际张量显存需求接近**
   - 峰值 allocated 显存都在 **9~10 GB** 左右
   - 平均 allocated 显存都在 **约 2 GB** 左右
   - 说明当前任务本身并不重度吃显存

4. **A100 的显存余量更大，扩展性更好**
   - 同样任务下，A100 的峰值 allocated 利用率仅 **11.43%**
   - B150S 达到 **29.51%**
   - 若后续提升 batch、context_length 或模型规模，A100 更有空间

5. **按总显存归一化后，B150S 的显存资源利用更紧凑**
   - 在当前任务没有吃满 A100 显存的前提下
   - B150S 的每 GiB 显存吞吐略优
   - 这说明 B150S 更像是“资源更紧但利用更满”，A100 则是“资源更充裕、速度更快”

6. **训练结果质量基本一致**
   - `train_loss` 几乎相同
   - 本次差异主要是**速度与资源使用特征差异**，不是明显的收敛质量差异

综合来看：

- 如果目标是 **更快完成 Chronos-2 训练任务、缩短实验周期、为更大任务保留显存余量**，A100 更合适。
- 如果目标是 **在较小显存资源下完成同等任务，并关注显存资源利用紧凑度**，B150S 也具备可用性，但训练时间明显更长。
